## 部署架构总览

```
用户浏览器
    │
    ▼
┌──────────────────────────────────────────────────┐
│                    Nginx (:80/:443)               │
│  ┌──────────────────┐  ┌───────────────────────┐  │
│  │ ai.youc.online   │  │ s2a.youc.online       │  │
│  │ (Sub2API 用户)   │  │ (S2A Manager 管理台)  │  │
│  └───────┬──────────┘  └───────────┬───────────┘  │
└──────────┼──────────────────────────┼──────────────┘
           │                          │
           ▼                          ▼
    127.0.0.1:8080             127.0.0.1:3001
    ┌──────────┐              ┌───────────────┐
    │ Sub2API  │              │ S2A Manager   │
    │ (不动)   │◄──Admin API──│ (新增)        │
    └──────────┘              └───────┬───────┘
                                      │
                              127.0.0.1:5432
                         ┌─────────────────────┐
                         │  PostgreSQL          │
                         │  ├─ sub2api 库(不动) │
                         │  └─ s2amanager 库(新)│
                         └─────────────────────┘
```

| 服务 | 域名 | 后端端口 | 说明 |
|------|------|---------|------|
| Sub2API | `ai.youc.online` | 8080 | 原样不动 |
| S2A Manager | `s2a.youc.online` | 3001 | 新增 |
| Worker | — | — | 后台定时任务 |

## 风险说明

### 🛑 绝对禁止

| 操作 | 后果 |
|------|------|
| `DROP DATABASE s2amanager;` | S2A Manager 全部数据丢失 |
| `npx prisma migrate reset` | S2A Manager 全部数据丢失 |
| `rm .env` | API Key 无法解密，所有连接配置失效 |
| 改 `ENCRYPTION_KEY` | 已保存的 Admin API Key 全部废掉 |
| 停 Sub2API 相关进程 | 用户服务中断 |

### ✅ 安全无影响

| 操作 | 说明 |
|------|------|
| 创建 s2amanager 数据库 | 独立新库，不碰已有数据 |
| `pnpm install` / `pnpm build` | 只在 S2A 目录内操作 |
| `npx prisma migrate deploy` | 只新增表/字段，不删数据 |
| systemd 新增服务 | 独立服务，不修改已有服务 |
| Nginx 新增 conf | 增量，不修改已有配置 |
| `certbot --nginx -d s2a.xxx` | 只给新域名加证书 |

## Step 0：环境检测

SSH 登录服务器后，先跑检测，确认哪些已有：

```bash
# 一键检测
echo "=== 环境检测 ===" && \
node --version 2>/dev/null && echo "✅ Node.js" || echo "❌ 缺 Node.js (v20+)" && \
pnpm --version 2>/dev/null && echo "✅ pnpm" || echo "❌ 缺 pnpm" && \
psql --version 2>/dev/null && echo "✅ PostgreSQL" || echo "❌ 缺 PostgreSQL" && \
pg_isready 2>/dev/null && echo "✅ PostgreSQL 运行中" || echo "❌ PostgreSQL 未运行" && \
ss -tlnp | grep -q ':3001' && echo "⚠️ 端口 3001 已被占用" || echo "✅ 端口 3001 空闲" && \
echo "=== 检测完毕 ==="
```

### 缺 Node.js（v20+）

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo bash -
sudo apt install -y nodejs
```

### 缺 pnpm + 配镜像源

```bash
sudo npm install -g pnpm
# 国内镜像加速
pnpm config set registry https://registry.npmmirror.com/
npm config set registry https://registry.npmmirror.com/
```

### 缺 PostgreSQL

```bash
sudo apt install -y postgresql postgresql-contrib
sudo systemctl enable --now postgresql
```

## Step 1：创建独立数据库

> ✅ 安全：只新建 s2amanager 库，不碰任何已有数据库。

```bash
sudo -u postgres psql
```

在 `postgres=#` 提示符下执行（把 `你的强密码` 换成实际密码）：

```sql
CREATE USER s2amanager WITH PASSWORD '你的强密码';
CREATE DATABASE s2amanager OWNER s2amanager;
GRANT ALL PRIVILEGES ON DATABASE s2amanager TO s2amanager;
\c s2amanager
GRANT ALL ON SCHEMA public TO s2amanager;
\q
```

验证：

```bash
psql -h 127.0.0.1 -U s2amanager -d s2amanager -c "SELECT version();"
# 输入密码后看到版本号 = 成功
```

## Step 2：克隆代码 + 配置 .env

```bash
cd /opt
sudo git clone https://github.com/Ganseblum/S2A-Manager.git s2a-manager
cd /opt/s2a-manager

# 生成密钥
openssl rand -base64 24   # → 填 APP_SECRET
openssl rand -base64 32   # → 填 ENCRYPTION_KEY

# 创建 .env
sudo cp .env.example .env
sudo nano .env
```

`.env` 内容（把生成的值填进去）：

```env
DATABASE_URL="postgresql://s2amanager:你的强密码@127.0.0.1:5432/s2amanager?schema=public"
APP_SECRET="openssl rand -base64 24 的输出"
ENCRYPTION_KEY="openssl rand -base64 32 的输出"
PORT=3001
```

> 🛑 **ENCRYPTION_KEY 生成后永远不能换！** 换了 = 已保存的 API Key 全部无法解密。

**立即备份 .env：**

```bash
sudo cp /opt/s2a-manager/.env /opt/s2a-manager/.env.backup
sudo cp /opt/s2a-manager/.env ~/s2a-env-$(date +%Y%m%d)
```

## Step 3：安装依赖 + 初始化 + 构建

### 3.1 修复 package.json 端口硬编码问题

```bash
sudo nano /opt/s2a-manager/package.json
```

把 `"start": "next start -p 3000"` 改为 `"start": "next start"`，保存退出。

> 不改的话 `.env` 里的 `PORT=3001` 会被覆盖，启动时会报 `EADDRINUSE`。

### 3.2 安装依赖

```bash
cd /opt/s2a-manager
sudo chown -R $(whoami):$(whoami) /opt/s2a-manager

# 安装所有依赖
pnpm install
```

### 3.3 补充注册 keychain

```bash
# 系统版本不同可能还会有遗漏，如果遇到再补
pnpm approve-builds @prisma/client prisma @prisma/engines esbuild unrs-resolver
pnpm install
```

### 3.4 初始化数据库表结构

```bash
pnpm prisma migrate deploy
```

### 3.5 构建

```bash
pnpm build
```

验证：

```bash
ls /opt/s2a-manager/.next/
# 有 BUILD_ID 等文件 = 构建成功
```

## Step 4：systemd 托管启动

> ✅ 安全：单独创建两个服务文件，不影响任何现有服务。

### 创建 Web 服务

```bash
sudo tee /etc/systemd/system/s2a-manager-web.service > /dev/null << 'EOF'
[Unit]
Description=S2A Manager Web
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/s2a-manager
Environment=NODE_ENV=production
Environment=PORT=3001
ExecStart=/usr/bin/pnpm start
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

### 创建 Worker 服务

```bash
sudo tee /etc/systemd/system/s2a-manager-worker.service > /dev/null << 'EOF'
[Unit]
Description=S2A Manager Worker
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/s2a-manager
Environment=NODE_ENV=production
ExecStart=/usr/bin/pnpm worker
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
```

### 启动并设开机自启

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now s2a-manager-web
sudo systemctl enable --now s2a-manager-worker
```

### 验证

```bash
# 两个服务都应该 active (running)
sudo systemctl status s2a-manager-web --no-pager -l
sudo systemctl status s2a-manager-worker --no-pager -l

# 端口 3001 应该 LISTEN
ss -tlnp | grep 3001
```

## Step 5：Nginx 反代 + HTTPS

### 5.1 添加 DNS 解析

去域名 DNS 管理后台，加一条 A 记录：

| 类型 | 主机记录 | 记录值 |
|------|---------|--------|
| A | `s2a` | 你的服务器 IP |

### 5.2 添加 Nginx 配置（增量，不动原有配置）

```bash
sudo tee /etc/nginx/conf.d/s2a-manager.conf > /dev/null << 'EOF'
server {
    listen 80;
    server_name s2a.youc.online;

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
    }
}
EOF
```

### 5.3 验证配置并重载

```bash
# 语法检查
sudo nginx -t

# 平滑重载（不影响 ai.youc.online 的已有连接）
sudo systemctl reload nginx
```

### 5.4 申请 SSL 证书（自动续期）

```bash
sudo certbot --nginx -d s2a.youc.online
```

- 选 **2（Redirect）**把 HTTP 自动跳转 HTTPS
- certbot 会自动加 systemd timer，证书到期前自动续期

验证自动续期：

```bash
sudo systemctl status certbot.timer
# 看到 Active: active (waiting) = 正常
```

## Step 6：浏览器关联 Sub2API

1. 打开 `https://s2a.youc.online`
2. 首次访问 → 创建管理员账号（填邮箱 + 密码，6 位以上）
3. 登录后 → 左侧菜单「添加连接」
4. 填写：

| 字段 | 填什么 |
|------|--------|
| 名称 | 随便起，如「我的 Sub2API」 |
| Sub2API 站点 URL | `http://127.0.0.1:8080`（填写你 Sub2API 实际端口） |
| Admin API Key | 在 Sub2API 管理后台获取 |

5. 点「**测试**」→ 显示「测试通过」= 联动成功
6. 点「**保存**」

### 验证联动

- 「分组倍率」页面能看到 Sub2API 的分组列表 = 读取正常
- 手动改一个测试分组倍率，点「保存并应用」→ 去 Sub2API 后台确认倍率更新 = 写入正常

> ⚠️ 倍率自动同步功能需要手动启用才会生效。先手动测通再开自动同步。

## 备份策略

### 备份 .env（最重要！）

```bash
sudo cp /opt/s2a-manager/.env ~/s2a-env-$(date +%Y%m%d)
```

### 备份数据库

```bash
sudo -u postgres pg_dump s2amanager | gzip > ~/s2amanager_$(date +%Y%m%d).sql.gz
```

### 恢复数据库

```bash
gunzip < ~/s2amanager_YYYYMMDD.sql.gz | sudo -u postgres psql s2amanager
```

### 自动备份脚本（可选）

```bash
sudo tee /root/backup-s2a.sh > /dev/null << 'SCRIPT'
#!/bin/bash
cp /opt/s2a-manager/.env /root/s2a-env-$(date +%Y%m%d)
sudo -u postgres pg_dump s2amanager | gzip > /root/s2amanager_$(date +%Y%m%d).sql.gz
find /root/s2amanager_*.sql.gz -mtime +30 -delete
SCRIPT
sudo chmod +x /root/backup-s2a.sh
```

添加 cron 每天自动备份：

```bash
(sudo crontab -l 2>/dev/null; echo "0 3 * * * /root/backup-s2a.sh") | sudo crontab -
```

## 升级操作

> 📋 **升级 Checklist**

### 完整升级流程

```bash
# ── 第 1 步：停服务（不影响 Sub2API）──
sudo systemctl stop s2a-manager-web s2a-manager-worker

# ── 第 2 步：备份（必做！）──
sudo cp /opt/s2a-manager/.env ~/s2a-env-$(date +%Y%m%d)
sudo -u postgres pg_dump s2amanager | gzip > ~/s2amanager_$(date +%Y%m%d).sql.gz

# ── 第 3 步：拉最新代码 ──
cd /opt/s2a-manager
git fetch upstream              # 如果 fork 了，拉原项目更新
git merge upstream/main         # 合入自己的 main 分支
# 或者直接 git pull origin main（如果没配 upstream）

# ── 第 4 步：安装新依赖 ──
pnpm install

# ── 第 5 步：数据库迁移（只新增，不删除）──
pnpm prisma migrate deploy

# ── 第 6 步：重新构建 ──
pnpm build

# ── 第 7 步：启动 ──
sudo systemctl start s2a-manager-web s2a-manager-worker

# ── 第 8 步：确认正常 ──
sudo systemctl status s2a-manager-web --no-pager -l
sudo systemctl status s2a-manager-worker --no-pager -l
ss -tlnp | grep 3001
curl -s -o /dev/null -w "%{http_code}" https://s2a.youc.online
```

### 升级失败回滚

```bash
# 停服务
sudo systemctl stop s2a-manager-web s2a-manager-worker

# 回滚代码（把 HASH 换成升级前的 commit）
cd /opt/s2a-manager
git checkout <升级前的 commit hash>

# 恢复数据库（如果 migrate 改了表结构）
gunzip < ~/s2amanager_YYYYMMDD.sql.gz | sudo -u postgres psql s2amanager

# 重装依赖 + 构建
pnpm install
pnpm build

# 启动
sudo systemctl start s2a-manager-web s2a-manager-worker
```

### 升级不需要动的部分

| 不需要动 | 原因 |
|---------|------|
| Nginx 配置 | 反代规则不变 |
| SSL 证书 | certbot 自动续期，和代码无关 |
| systemd 配置 | 服务定义不变 |
| `.env` | 数据库、密钥不变 |
| DNS 解析 | 域名不变 |


## 常见问题

### Q：S2A Manager 会影响 Sub2API 的用户吗？

不会。S2A Manager 只通过 Admin API 操作，且倍率同步需手动启用。默认不会改 Sub2API 的任何数据。

### Q：S2A Manager 挂了，Sub2API 还能用吗？

能。两个服务完全独立，S2A Manager 挂了只是管理后台打不开，Sub2API 照常服务。

### Q：升级时 Sub2API 需要停机吗？

不需要。升级过程只停 S2A Manager 自己的 systemd 服务，Sub2API 全程在线。

### Q：Nginx reload 会影响线上服务吗？

不会。`reload` 是平滑重载，已有连接不断开，新请求秒切新配置。而且新加的是一个独立 server 块，不动原有配置。

### Q：.env 里的 ENCRYPTION_KEY 丢了呢？

需要重新生成 Key，但之前保存的所有 Sub2API 连接中的 Admin API Key 无法解密，必须重新添加连接。

### Q：worker 日志里的 "Balance alerts skipped" 是什么？

正常日志。表示余额告警还没到触发时间，worker 跳过本次检查。每 10 分钟出现一次是正常的。
