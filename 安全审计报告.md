## 一、审计结论

**未发现任何数据上报、泄露、后门或同步到第三方地址的行为。**

项目所有网络请求都只发往用户自己配置的目标地址（Sub2API 站点、采集源站、Webhook URL），不存在任何遥测、分析、追踪或第三方上报代码。

## 二、网络请求出口分析

### 唯一的 HTTP 请求函数

整个项目中只有 **一个** `fetch()` 调用，位于 `src/server/http.ts:194`：

```typescript
const res = await fetch(options.url, { ... });
```

所有对外请求都经过这个 `requestText()` 函数。URL 由调用方传入，**无任何硬编码的远程地址**。

### 所有请求目标一览

| 请求发起方 | 目标地址 | 来源 |
|---|---|---|
| `Sub2ApiAdminClient` | `${connection.baseUrl}/api/v1/admin/*` | 用户在连接管理中配置的 Sub2API 站点 URL |
| `BlSub2ApiClient` | `${site.baseUrl}/api/v1/...` | 用户配置的采集源站 URL |
| `BlNewApiClient` | `${site.baseUrl}/api/...` | 用户配置的采集源站 URL |
| `sendWebhook()` (余额预警) | 用户配置的 Webhook URL | 用户在账号余额预警中配置的 URL |
| 前端 tRPC 请求 | `/api/trpc` (相对路径) | 本机同源请求 |

### 前端请求分析

前端通过 tRPC + React Query 与后端通信，tRPC 客户端配置为：

```typescript
httpBatchLink({ url: "/api/trpc", transformer: superjson })
```

**所有前端请求都是相对路径 `/api/trpc`，即同源请求，不涉及任何外部地址。**

前端代码中的 `refetch()` 调用是 React Query 的数据刷新方法，本质是重新发起 tRPC 请求到 `/api/trpc`，不是外部 HTTP 请求。

## 四、敏感数据保护分析

### 4.1 Admin API Key

| 环节 | 保护措施 |
|---|---|
| 存储 | AES-256-GCM 加密后存入数据库 `connections.admin_api_key` 字段 (`crypto.ts`) |
| 解密 | 仅在需要调用 Sub2API Admin API 时临时解密，使用后不持久化 |
| 前端返回 | `maskConnection()` 函数将 `adminApiKey` 替换为 `••••••`（`connections.ts`），**明文永不返回前端** |
| 传输 | 仅通过本机 tRPC 同源请求传输，不经过第三方 |

### 4.2 管理员密码

| 环节 | 保护措施 |
|---|---|
| 存储 | bcrypt 哈希（cost factor = 12），存入 `admin_users.password_hash` |
| 登录验证 | `attemptLogin()` 使用 `bcrypt.compare()` 比对 |
| 前端返回 | `listUsers()` 只返回 `id, email, createdAt`，**密码哈希不返回** |

### 4.3 采集源站密码

| 环节 | 保护措施 |
|---|---|
| 存储 | AES-256-GCM 加密后存入 `bl_collection_sites.password_enc` |
| 解密 | 仅在登录采集源站时临时解密 |

### 4.4 会话安全

| 环节 | 保护措施 |
|---|---|
| 签发 | JWT (HS256)，密钥为环境变量 `APP_SECRET` |
| Cookie | `httpOnly` + `secure` + `sameSite=lax`，7 天过期 |
| 验证 | middleware.ts 在路由层验证 JWT，`protectedProcedure` 在 tRPC 层二次验证 |
| 注销 | `clearSessionCookie()` 清除 cookie |

### 4.5 SQL 注入防护

- 所有数据库查询通过 Prisma ORM 执行
- `data.ts` 中的 raw SQL 使用 `Prisma.sql` 模板字面量，参数自动转义
- 无字符串拼接 SQL

## 五、`new Function()` 沙箱详细分析

位置：`src/server/bl-bindings.ts:387`

```typescript
const fn = new Function(...argNames, `"use strict"; return (${expression});`);
```

**保护措施**：

1. **白名单校验**：表达式在执行前经过 `validateCustomExpression()` 校验，只允许以下字符和关键词：
   - 数字、加减乘除、括号
   - `avg`, `min`, `max`, `round`, `floor`, `ceil`, `abs`, `sum`, `count` 函数
   - 空格和逗号
2. **无全局作用域访问**：`new Function` 创建的函数不闭包局部变量，只能访问传入的参数和全局对象
3. **参数受限**：`argNames` 是固定的 `["avg", "min", "max", "round", "floor", "ceil", "abs", "sum", "count", "currentRate", ...sourceRates]`
4. **使用严格模式**：`"use strict"` 防止意外创建全局变量

**风险评估**：安全。用户无法通过自定义公式访问 `process`, `require`, `fetch` 等危险 API，因为白名单正则会在执行前拒绝任何非数学表达式。

## 七、总结

1. **无数据上报**：项目不含任何遥测、分析、追踪或错误上报 SDK
2. **无后门**：无硬编码的远程地址，所有请求目标由用户配置
3. **无数据泄露**：Admin API Key 加密存储 + 前端脱敏，密码 bcrypt 哈希
4. **所有请求都是用户可控的**：请求发往用户自己配置的 Sub2API 站点、采集源站或 Webhook URL
5. **前端请求全部同源**：前端只通过 `/api/trpc` 相对路径与后端通信
6. **唯一的 GitHub 链接**是可点击的仓库引用，不是数据上报

**该项目可以安全部署使用，不会将用户的 Sub2API 配置、Admin API Key 或任何运营数据泄露给第三方。**
