## 风险说明

### 🛑 高风险操作（禁止随意执行）

| 操作 | 命令 | 后果 | 替代方案 |
|------|------|------|---------|
| 删除数据库 | `DROP DATABASE s2amanager;` | **全部数据永久丢失** | 用备份恢复 |
| 重置数据库 | `npx prisma migrate reset` | **全部数据永久丢失** | 永远不要用在生产/有数据的环境 |
| 删除 .env | `rm .env` | API Key 无法解密，连接配置全部失效 | 先备份再操作 |
| 更换 ENCRYPTION_KEY | 修改 `.env` 后重启 | 已保存的 Admin API Key 全部无法解密 | 永远不要更换 |
| 删除 Sub2API 目录 | `rm -rf sub2api/` | **Sub2API 全部数据丢失** | 不要执行 |
| 停止 Sub2API 服务 | `npm run stop` / `pkill` | **服务停机，用户无法使用** | 选低峰期操作 |

### ✅ 安全操作

| 操作 | 说明 |
|------|------|
| `pnpm install` | 只安装依赖，不修改数据 |
| `pnpm prisma migrate deploy` | 只新增表/字段，不删除现有数据 |
| `npm run build` | 只构建前端，不修改数据 |
| `cp -a` 备份 | 只读操作，安全 |
| `pg_dump` 备份 | 只读操作，安全 |

## Step 2：创建 PostgreSQL 数据库

> ✅ 安全：此步骤只创建新数据库，不影响已有数据。
> 💡 如果 Step 0 检测显示「数据库 s2amanager 已存在」，**跳过此步**。

```bash
# 登录 PostgreSQL（macOS Homebrew 安装的不需要密码，直接进）
psql postgres

# 在 PostgreSQL 提示符下执行：
CREATE USER s2amanager WITH PASSWORD '你的密码';
CREATE DATABASE s2amanager OWNER s2amanager;
GRANT ALL PRIVILEGES ON DATABASE s2amanager TO s2amanager;
\c s2amanager
GRANT ALL ON SCHEMA public TO s2amanager;
\q
```

> 把 `你的密码` 替换为一个强密码，记下来。

验证数据库创建成功：

```bash
psql postgres -U s2amanager -d s2amanager -c "SELECT version();"
```

## Step 3：配置 S2A Manager

> 💡 如果 Step 0 检测显示「.env 已存在」，**跳过此步**。

```bash
cd /Users/hang/Documents/myProject/S2A-Manager

# 生成密钥
openssl rand -base64 24  # 复制输出，填 APP_SECRET
openssl rand -base64 32  # 复制输出，填 ENCRYPTION_KEY

# 创建 .env
cp .env.example .env
nano .env
```

`.env` 内容（替换实际值）：

```env
DATABASE_URL="postgresql://s2amanager:你的密码@127.0.0.1:5432/s2amanager?schema=public"
APP_SECRET="这里填 openssl rand -base64 24 的输出"
ENCRYPTION_KEY="这里填 openssl rand -base64 32 的输出"
PORT=3001
```

> ⚠️ **ENCRYPTION_KEY 生成后绝对不能改！** 建议立即备份：
> ```bash
> cp .env ~/.s2a-manager-env-backup
> ```

## Step 5：关联 Sub2API

这是最关键的一步，让 S2A Manager 能管理你的 Sub2API。

### 5.1 获取 Sub2API Admin API Key

在 Sub2API 管理后台（即 Sub2API 的 Web 界面）生成 Admin API Key。

### 5.2 在 S2A Manager 添加连接

1. 登录 S2A Manager（`http://localhost:3001`）
2. 点左侧「添加连接」或首页的「添加连接」
3. 填写：
   - **名称**：如「我的 Sub2API」
   - **Sub2API 站点 URL**：`http://127.0.0.1:3000`（如果 Sub2API 在同一台且用默认端口）
   - **Admin API Key**：填写上一步获取的 Key
4. 点「测试」→ 显示「测试通过」说明连通正常
5. 点「保存」

### 5.3 验证联动

添加连接后，S2A Manager 就能：
- 读取 Sub2API 的分组列表 → 在「分组倍率」页面可以看到
- 读取 Sub2API 的账号列表 → 在「账号调度」页面可以看到
- 通过 Admin API 修改倍率 → 启用倍率规则后自动写入

> ⚠️ **第一次建议先手动测试**：
> 1. 进「分组倍率」页面
> 2. 选一个测试分组
> 3. 绑定采集源（如果没有采集源，先跳过）
> 4. 点「保存并应用」
> 5. 去 Sub2API 管理后台确认倍率是否正确写入

## 备份策略

### 备份 .env（最重要！）

```bash
cp /Users/hang/Documents/myProject/S2A-Manager/.env ~/.s2a-manager-env-backup
```

> `ENCRYPTION_KEY` 丢失 = 所有已保存的 Admin API Key 无法解密 = 需要重新填写所有连接

### 备份数据库

```bash
# 手动备份
pg_dump -h 127.0.0.1 -U s2amanager s2amanager | gzip > ~/s2amanager_backup_$(date +%Y%m%d).sql.gz

# 恢复（需要时执行）
gunzip < ~/s2amanager_backup_YYYYMMDD.sql.gz | psql -h 127.0.0.1 -U s2amanager s2amanager
```

### 备份项目目录（升级前必做）

```bash
cp -a /Users/hang/Documents/myProject/S2A-Manager /Users/hang/Documents/myProject/S2A-Manager.backup.$(date +%Y%m%d_%H%M%S)
```

## 常见问题

### Q：Sub2API 和 S2A Manager 能同时运行吗？

能，只要端口不同。Sub2API 用 3000，S2A Manager 用 3001，互不干扰。

### Q：S2A Manager 会影响 Sub2API 的现有用户吗？

不会。S2A Manager 只通过 Admin API 操作，不可以直接修改 Sub2API 的文件或数据库。倍率同步功能需要你手动启用才会生效。

### Q：如果 S2A Manager 的数据库坏了，Sub2API 还能用吗？

能。两个数据库完全独立。S2A Manager 的 PostgreSQL 数据库坏了，重新部署一个就行，Sub2API 不受影响。

### Q：怎么确认 S2A Manager 和 Sub2API 真的联动了？

在 S2A Manager 的「分组倍率」页面能看到 Sub2API 的分组列表，说明读取正常。手动改一个分组的倍率并「保存并应用」，然后去 Sub2API 管理后台确认倍率已更新，说明写入正常。
